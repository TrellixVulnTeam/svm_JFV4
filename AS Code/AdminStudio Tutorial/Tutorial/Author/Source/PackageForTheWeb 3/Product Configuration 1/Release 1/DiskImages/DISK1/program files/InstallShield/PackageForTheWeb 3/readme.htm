Introduction

Thank you for purchasing InstallShield's PackageForTheWeb Internet distribution utility.  PackageForTheWeb is designed to give you the easiest and most efficient route to putting your installations and files on the web.

This document is intended to give you a brief overview of some of the more important aspects about using PackageForTheWeb and being an InstallShield customer that you will need to know before you start creating your own PackageForTheWeb packages. 

Digital Signing

  • If you wish to use Microsoft® AuthenticodeTM technology to digitally sign your package, you must have Microsoft Internet Explorer Version 3.0x  with the Authenticode 2.0 Update, or Internet Explorer 4.0 or later installed on your system.

  • If you do not have an Authenticode signature and would like to get one, you can visit the Verisign Web site for more information.

  • To obtain the AuthenticodeTM 2.0 Update or to check whether you need the update, please visit Microsoft's Internet Explorer Security Web site.

  • Because the AuthenticodeTM 2.0 code signing tools included with PackageForTheWeb must match the browser environment that they are run on, upgrading your browser from Internet Explorer 3.0x to Internet Explorer 4.0x may break the ability of PackageForTheWeb to digitally sign your files. If you decide to upgrade your browser, please reinstall PackageForTheWeb as that will ensure you will have the correct version of these code signing tools.
