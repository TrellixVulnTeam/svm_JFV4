Deleting Packages

AdminStudio Enterprise Server 8.0 Â» SMS Web Console

From the SMS Web Console, you can delete a package from the Application Catalog, from the SMS Server, or both.

Caution

Deleting a distributed package from SMS will also delete its programs and any advertisements of the programs. If the package has source files, they will be removed from distribution points. If there are access accounts for this package, they will be deleted. Also, SMS Administrators' security rights to the package will be deleted.

Caution

When you delete a package from the Application Catalog, all package configuration information is lost, even if you do not delete it from SMS. If you delete a package from the Application Catalog but not from SMS, the package would still be available on the SMS Server, but all connection between the Application Catalog and SMS is lost. Should you want to distribute that package again in the future, you would have to re-import the application into the Application Catalog, and then re-enter package configuration information on the SMS Package Configuration Page.

Task

To delete a package:

  1. Launch SMS Web Console. The SMS Web Console Home Page opens.
  2. Click Package Administration. The Package Administration Page appears, listing all packages in your Application Catalog. In the Status column, each package's status is either Distributed or Not Distributed.
  3. Select a package and click Delete Distribution. The SMS Server Connection Settings Page appears.
  4. Enter a valid User Name and Password for the SMS Server you want to connect to, and click Next. The Confirm Delete Page appears prompting you to confirm that you want to delete this package from the Application Catalog. If you are deleting a Distributed package, you are also prompted to confirm that you want to delete this package from SMS.
  5. Select Yes or No to indicate your deletion preferences:

    Package Status

    Delete from Catalog?

    Delete from SMS?

    Not Distributed

    Choose Yes to confirm the deletion from the Application Catalog or choose No to decline the deletion.

    If you choose Yes, the package will be deleted from the Application Catalog, along with all of the package configuration information that you entered on the SMS Package Configuration Page.

    N/A

    Distributed

    Choose Yes to confirm the deletion from the Application Catalog, or choose No to decline the deletion.

    If you delete a package from SMS but do not delete it from the Application Catalog, all of the package configuration information that you entered on the SMS Package Configuration Page is saved, making it easy to distribute this package again in the future.

    Choose Yes to confirm the deletion from SMS, or choose No to decline the deletion.

    Unless you are certain that you will never want to distribute this package again, it is recommended that if you delete a package from SMS, you do not delete it from the Application Catalog.

  6. Click OK to confirm your selections.
